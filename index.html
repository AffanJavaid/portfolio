Staging Environment Optimization with Karpenter

Designed a cost-efficient staging environment using Karpenter on EKS with exclusive spot instances and strict scaling limits to balance performance and cost.

Karpenter EKS Spot Instances Terraform Cost Control
Key Implementation:
  • 100% spot instances for staging with capacity-optimized strategy
  • Set 50GB memory limit for cluster-wide scaling
  • Configured instance type flexibility (10+ instance types)
  • Implemented auto-scaling based on memory requests
  • Saved 70% costs compared to on-demand instances
Terraform Snippet:
resource "karpenter_provisioner" "staging" {
            name = "staging-spot"
            requirements {
                key      = "karpenter.sh/capacity-type"
                operator = "In"
                values   = ["spot"]
            }
            limits {
                resources {
                memory = "50Gi"
                }
            }
            provider_ref {
                name = "aws"
            }
            }
